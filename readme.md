# Hotspot Detection

ทำหน้าที่หาหัวไฟเพื่อช่วยสนุนผู้ใช้งานในการสร้างข้อมูลเข้าให้กับ forefire ที่จะทำนายการงามของไฟ

## Route (Restful API, JSend)
สำหรับให้ express มาเรียกใช้งานเท่านั้น
- GET `/docs` Swagger (auto-generated by FastAPI)
- GET `/api/v1/hotspot-detection` ดู instance ทั้งหมด
- POST `/api/v1/hotspot-detection` สร้าง instance
- GET `/api/v1/hotspot-detection/:id/snapshot`
- DELETE `/api/v1/hotspot-detection/:id` ลบ instance
- DELETE `/api/v1/hotspot-detection` ลบ instance ทั้งหมด

## Service (instance)
- instance จะมีอายุแค่ 10 นาที (กันลืมปิด)
- งานที่ทำ (loop)
    1. ดึงภาพจาก MediaMTX (cv2 with ffmpeg backend)
    2. ตรวจจับ Hotspots (yolo)
    3. วาดกรอบ + ข้อความ (cv2)
    4. ส่งภาพไปยัง MediaMTX ใน path ใหม่ เช่น AI/drone_01 ผ่าน ffmpeg

## ข้อควรระวัง
- ให้เก็บ PID ของ subprocess ไว้ใน instance ด้วย (ตอบมันตายเราจะได้ clear ทั้งหมดได้ง่าย)
- ตอนเอาไปใช้อย่าลืม app/weights/best.pt
